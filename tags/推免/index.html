    操作系统的作用

    1. 用户与计算机硬件系统之间的接口。
      用户在OS帮助下方便快捷地操纵计算机硬件和运行自己的程序。

    2. 计算机系统资源的管理者。
      处理机、存储器、I/O设备以及文件（程序和数据）四类资源的有效管理。

    3. 操作系统实现了对计算机资源的抽象。
      隐藏了对硬件操作的具体细节。

    进程同步

    三个问题：

    1. 一个进程如何把信息传递给另一个。

    2. 如何确保两个或多个进程在关键活动中不会出现交叉。

    3. 正确的信息传递顺序。

    两个或多个进程读写某些共享数据，最后的结果取决于进程运行的精确时序称为竞争条件。多核增长带来的并行使得竞争条件越来越普遍。

    避免竞争条件需要互斥。

    把对共享内存进行访问的程序片段称为临界区，要保证使用共享数据的并发进程能够正确和高效地进行协作，同步机制应遵循的规则：

    1. 空闲让进。无进程处于临界区使允许请求的进程进入。

    2. 忙则等待。当已有进程进入时，其他试图进入临界区的进程必须等待。

    3. 有限等待。对要求访问临界区的进程，应该保证有限时间内进入。

    4. 让权等待。当进程不能进入临界区时，应立即释放处理及，避免进入“忙等”。

    同步机制

    1. 关中断

    2. Test-and-Set指令

    3. Swap指令

    4. 信号量

      4.1. 整型信号量

      4.2. 记录型信号量，用链表记录等待进程

      4.3. AND型信号量，同时请求多类资源1个

      4.4. 信号量集，同时请求多类资源多个

    5. 管程

      利用共享数据结构抽象地表示系统中的共享资源，并将对该共享数据结构实施的特定操作定义为一组过程，包含了面向对象的思想。任一时刻管程中只能有一个活跃进程。

    进程间通信

    进程通信是指进程之间的信息交换，由于进程的互斥与同步，需要在进程间交换一定的信息，故不少学者将他们也归为进程通信，但只能把它们称为低级进程通信。之所以低级是因为：

    1. 效率低，生产者/上传者每次只能从缓冲区取一个消息。

    2. 通信对用户不透明，OS只为进程间通信提供了共享存储器，通信所需的数据结构、数据传送、互斥与同步都必须由程序员实现。

    在进程之间要传送大量数据时，应当使用OS提供的高级通信工具，该工具最主要的特点是：

    1. 使用方便。OS隐藏了实现的具体细节，向用户提供了一组用于实现高级通信的命令（原语）。通信过程对用户是透明的。

       透明性：本来存在的事物从某个角度看似乎不存在。类似的有计算机网络中底层对高层是透明的。

    2. 高效地传送大量数据。用户可以直接利用高级通信命令（原语）高效地传送大量数据。

    进程间通信的类型

    1. 共享存储器系统。又可分为共享数据结构（适用于传递少量数据，属于低级通信，例如生产者-消费者问题中的有界缓冲区）或共享存储区（需要通信的进程在通信时向系统请求获得共享存储区中的一个分区，将其附加到自己的地址空间中，对其中数据进行读写，不再需要时返还）。

       地址空间：多任务操作系统中的每一个进程都运行在一个属于它自己的内存沙盘中，这个沙盘就是虚拟地址空间（virtual address space），在32位模式下，它总是一个4GB的内存地址块。这些虚拟地址通过页表（page table）映射到物理内存，页表由操作系统维护并被处理器引用。

    2. 管道（pipe）通信系统
      管道指用于连接一个读进程和一个写进程以实现它们之间通信的一个共享文件，又名pipe文件。写进程以字符流形式将大量数据送入管道，读进程从管道中接收数据。

    3. 消息传递系统
      以格式化的消息为单位，将通信的数据封装在消息中，利用操作系统提供的一组通信命令（原语），在进程间进行消息传递，完成数据交换。

    4. 客户机-服务器系统

      4.1 套接字Socket
      一个套接字就是一个通信标识类型的数据结构，包含里目的地址、端口号、传输层协议、进程所在的网络地址等。（网络中的套接字=IP+端口）
      4.2 远程过程调用和远程方法调用
      该协议允许运行于一台主机的系统上的进程调用另一台主机上的进程，对程序员表现为常规的过程调用，无需额外地为此编程。如果涉及面向对象编程，也可以叫远程方法调用。

    进程与线程的比较

    1. 线程是调度的基本单位，进程是操作系统分配资源的基本单位。

    2. 线程和进程都可以并发，不同进程可以并发执行，同一个进程的多个线程也可以。

    3. 每个进程都拥有独立的地址空间，不允许其他进程访问。而同一个进程的不同线程共享进程的地址空间和资源。

    4. 进程创建和撤销、上下文切换的开销明显大于线程。

    死锁

    一组进程中的每个进程都在等待仅由该组进程中其他进程才能引发的事件。

    引起死锁的情况：

    1. 竞争不可抢占资源。
    2. 竞争可消耗资源。
    3. 进程推进顺序不当。

    产生死锁的条件：

    1. 互斥条件。一段时间内某资源只能被一个进程使用。
    2. 请求和保持条件。进程已经保持了至少一个资源，并请求新的资源。
    3. 不可抢占条件。进程已获得的资源在未使用完之前不能被抢占。
    4. 循环等待条件。在发生死锁时，必然存在一个进程——资源的循环链。

    预防死锁：
    破坏以上四个条件中任意一个。

    分页、分段存储管理

    页是信息的物理单位，分页是系统的敬畏，对用户不可见。段是信息的逻辑单位，通常包含一组意义相对完整的信息。分段的目的在于能更好地满足用户的需要。

    页的大小固定且由系统决定。段的长度不固定，决定于用户所编写的程序。

    分页的用户程序地址空间是一维的线性地址空间，分段是用户的行为，在分段系统中，用户程序的地址空间是二维的，段号和段内地址。
